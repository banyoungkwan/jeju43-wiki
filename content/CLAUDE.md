# 제주4·3 아카이브 위키 — Schema & Operating Manual

이 파일은 LLM(Claude)이 이 wiki를 관리하는 방법을 정의한 스키마다.
모든 작업 전에 이 파일을 반드시 먼저 읽고, 아래 규칙과 워크플로를 따르라.

## 0. 이 Wiki의 목적

제주4·3(1947-1954)에 관한 학술 공개 아카이브 위키.
1차 사료(진상조사보고서, 미군정 문서, 증언 자료집)와 2차 연구를 축적하여
연구자·시민 누구나 접근 가능한 공개 지식베이스를 구축한다.

**이중언어 원칙**: 모든 개념·엔티티 페이지는 한국어와 영어를 병행 수록한다.
영어권 연구자에게 제대로 된 학술 정보를 제공하는 것이 이 위키의 핵심 목적 중 하나다.

Quartz로 정적 웹사이트(https://banyoungkwan.github.io/jeju43-wiki)로 발행된다.

## 2. 페이지 컨벤션

### 2-1. 이중언어 페이지 구조 (개념·엔티티 필수)

모든 `wiki/concepts/` 및 `wiki/entities/` 페이지는 아래 형식을 따른다.

**YAML 프론트매터**:
```yaml
---
title: 한국어 제목 / English Title
aliases: [한국어별칭, English alias, 한자표기]
tags: [태그1, 태그2]
lang: bilingual
created: YYYY-MM-DD
updated: YYYY-MM-DD
status: draft | review | stable
sources:
  - sources/소스페이지명
---
```

**본문 구조**:
```markdown
# 한국어 제목 / English Title

## 한국어

[한국어 본문 — 학술적 서술, ### 하위 섹션, [^n] 각주]

---

## English

[English content — academic register, ### subsections, [^n] footnotes]

[^1]: 출처 정보

## 관련 항목 / See Also

- [[위키링크]]
```

**규칙**:
- `title` 필드: `한국어 / English` 형식 (슬래시 양쪽 공백)
- `lang: bilingual` 필드 필수
- 한국어·영어 섹션은 `---` 구분선으로 분리
- 영어 섹션은 학술 영어 레지스터 유지; 한국어 원어는 괄호 또는 이탤릭 병기
- 각주 `[^n]`은 한국어·영어 섹션에 각각 독립 사용 (공유 가능)
- 하위 섹션은 `###` (h3) 사용; h4 이하는 가급적 지양

### 2-2. 소스 페이지 구조 (sources/)

소스 페이지는 한국어 기본. 원문이 영어인 경우 영어 원문 인용 허용.

**YAML 프론트매터**:
```yaml
---
title: "[사료] 문서 제목"
tags: [사료, 관련태그]
type: primary | secondary
created: YYYY-MM-DD
updated: YYYY-MM-DD
lang: ko | en | bilingual
---
```

**본문 구조 (방대한 자료의 경우)**:
방대한 문서(100페이지 이상)는 단순 요약에 그치지 않고,
주제별·시기별로 상세히 분화하여 기술한다:

```markdown
## 기본 정보
(발행기관, 발행일, 분량, 목차)

## 주요 내용 — 시기별 / 주제별
### [시기 또는 주제 1]
### [시기 또는 주제 2]
...

## 핵심 통계 및 수치
## 핵심 인용구
## 기존 자료와의 비교·쟁점
## 이 자료에서 도출된 wiki 항목
```

### 2-3. 파일명 규칙

- 파일명: 한국어 또는 영어, 하이픈 구분
- 예: `이행기정의.md`, `9연대.md`, `진상조사보고서.md`
- 소스 페이지: 원본 파일명 기반. 예: `화해와상생.md`

## 3. 소스 인제스트 워크플로

1. **소스 확인**: 파일명과 유형(1차/2차 사료) 확인
2. **읽기**: raw/ 경로 또는 kordoc 변환 후 읽기
3. **논의**: 핵심 내용과 기존 wiki와의 연결점 확인
4. **소스 요약 페이지 생성**: `sources/[카테고리]/[filename].md`
   - 방대한 자료: 주제·시기별로 상세히 분화 (단순 요약 금지)
5. **관련 페이지 업데이트**: 개념·인물·사건 페이지 갱신 또는 신규 생성
   - 새로운 개념·엔티티가 필요하면 적극적으로 생성
   - 충돌 주장: `> 쟁점:` 블록으로 표시
6. **index.md 업데이트**
7. **index-internal.md 업데이트**
8. **log.md 기록**: `## [YYYY-MM-DD] ingest | 소스 제목`

## 4. 쿼리 워크플로

1. index-internal.md로 관련 페이지 파악
2. 관련 페이지 읽고 종합
3. [[page]] 링크 포함한 답변
4. 유의미한 분석은 analyses/ 저장 제안
5. log.md에 `## [YYYY-MM-DD] query | 질문 요약` 기록

## 5. 린트 워크플로 (lint 명령 시)

- 고아 페이지 (인바운드 링크 없음)
- 충돌 주장 미표시
- 구식이 된 주장
- 필요하나 없는 개념/인물 페이지
- 누락된 크로스레퍼런스
- 연표 공백
- 이중언어 누락 (`lang: bilingual` 없는 개념·엔티티 페이지)
