# 제주4·3 아카이브 위키 — Schema & Operating Manual

이 파일은 LLM(Claude)이 이 wiki를 관리하는 방법을 정의한 스키마다.
모든 작업 전에 이 파일을 반드시 먼저 읽고, 아래 규칙과 워크플로를 따르라.

## 0. 이 Wiki의 목적
제주4·3(1947-1954)에 관한 학술 공개 아카이브 위키.
1차 사료(진상조사보고서, 미군정 문서, 증언 자료집)와 2차 연구를 축적하여
연구자·시민 누구나 접근 가능한 공개 지식베이스를 구축한다.
Quartz로 정적 웹사이트(https://banyoungkwan.github.io/jeju43-wiki)로 발행된다.

## 1. 디렉토리 구조
content/
├── CLAUDE.md               ← 이 파일 (스키마, 발행 제외)
├── index.md                ← 홈페이지 (발행됨)
├── index-internal.md       ← LLM 전용 카탈로그 (발행 제외)
├── log.md                  ← 작업 이력 (발행 제외)
├── wiki/
│   ├── concepts/           ← 개념 페이지
│   ├── entities/           ← 인물·기관·사건·장소 페이지
│   ├── sources/            ← 소스 요약 페이지
│   ├── analyses/           ← 심층 분석·비교
│   └── timelines/          ← 연표·시계열 분석
├── raw/                    ← 원본 소스 참조 (발행 제외)
└── templates/              ← 템플릿 (발행 제외)

발행 제외: CLAUDE.md, log.md, index-internal.md, raw/, templates/

## 2. 페이지 컨벤션
파일명: 소문자 영어 또는 한국어, 하이픈 구분. 예: april-third.md, lee-seungman.md
YAML 프론트매터 (필수): title, type, tags, date_range, created, updated, sources, related
언어: 한국어 기본, 학술 용어 원어 병기
인용: [^1] 각주 형식, 1차 사료는 [사료] 기관명, 문서제목, 날짜

## 3. 소스 인제스트 워크플로
1. 소스 확인: 파일명과 유형(1차/2차 사료) 확인
2. 읽기: raw/ 경로 또는 첨부 파일 읽기
3. 논의: 핵심 내용과 기존 wiki와의 연결점 확인
4. 소스 요약 페이지 생성: wiki/sources/[filename].md
5. 관련 페이지 업데이트: 개념·인물·사건 페이지 갱신 또는 신규 생성
   - 충돌 주장: > 쟁점: 블록으로 표시
6. index.md 업데이트
7. index-internal.md 업데이트
8. log.md 기록: ## [YYYY-MM-DD] ingest | 소스 제목

## 4. 쿼리 워크플로
1. index-internal.md로 관련 페이지 파악
2. 관련 페이지 읽고 종합
3. [[page]] 링크 포함한 답변
4. 유의미한 분석은 wiki/analyses/ 저장 제안
5. log.md에 ## [YYYY-MM-DD] query | 질문 요약 기록

## 5. 린트 워크플로 (lint 명령 시)
- 고아 페이지 (인바운드 링크 없음)
- 충돌 주장 미표시
- 구식이 된 주장
- 필요하나 없는 개념/인물 페이지
- 누락된 크로스레퍼런스
- 연표 공백
